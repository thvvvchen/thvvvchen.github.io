JS 闭包深度解析 —— 从作用域到内存回收

Jan 01, 2026
阅读时长: 3 分钟

1.2.1 闭包的形成原理

1. 作用域与作用域链

JavaScript 采用词法作用域（Lexical Scope）规则：

函数的作用域在定义阶段就已确定，而不是在调用阶段。

当一个函数被创建时，JS 引擎会：

  • 为该函数保存一份外部词法环境的引用
  • 形成一条从当前环境指向外部环境的 作用域链
1

当前执行环境 → 外层函数环境 → 全局环境

2. 闭包的本质定义

闭包并不是某种特殊语法，而是一种结果：

函数 + 创建时所捕获的词法环境 = 闭包

只要函数能够在其定义环境之外访问外部变量，这个函数就形成了闭包。

3. 闭包形成的必要条件

闭包通常需要同时满足以下三个条件：

  1. 函数嵌套
  2. 内部函数引用外部函数变量
  3. 内部函数在外部函数作用域之外被调用

function outer() {
  let x = 10;

const fn = outer();

outer 已经执行完毕，但 inner 仍然可以访问 x —— 这正是闭包存在的体现。

1.2.2 内存模型与生命周期

1. JavaScript 的堆与栈模型

在 JS 中，内存主要分为两类：

栈（Stack）

  • 存储：

    • 基本类型（number、string、boolean、null、undefined、symbol）
    • 执行上下文（Execution Context）

  • 特点：

    • 先进后出
    • 生命周期短
    • 自动回收

堆（Heap）

  • 存储：

    • 引用类型（Object、Array、Function）

  • 特点：

    • 动态分配
    • 需要垃圾回收机制管理

函数对象本身存在于堆中，而执行上下文在栈中

2. 活动对象（AO）与作用域链

当函数执行时，会创建一个执行上下文，其中包含一个 活动对象（AO）：

  • AO 内部存储：

    • 形参
    • 局部变量
    • 函数声明
    • this

多个执行上下文通过引用关系构成作用域链。

3. 闭包下的生命周期变化

闭包对内存生命周期的最大影响在于：

它打破了“函数执行完即可销毁变量”的直觉。

执行流程拆解

  1. 外部函数调用
    • 创建 AO
    • AO 入作用域链
  1. 内部函数创建
    • 内部函数对象被分配到堆内存
    • 保存对当前作用域链的引用
  1. 外部函数执行结束
    • 执行上下文出栈
    • ⚠️ 但 AO 仍被内部函数引用 → 无法回收
  1. 内部函数执行
    • 通过保存的作用域链访问外部变量

3
4
5

堆（函数对象）
   ↓
作用域链
   ↓
外部 AO（未被销毁）

这正是闭包“持有外部变量”的根本原因。

1.2.3 闭包的经典应用场景

1. 数据私有化（封装状态）

function createCounter() {
  let count = 0;
  return {
    increment: () => ++count,
    decrement: () => --count,
    getCount: () => count
  };
}

const counter = createCounter();
counter.increment();
counter.getCount(); // 1
  • count 不暴露到全局
  • 只能通过受控 API 操作
  • 是 JS 实现“私有变量”的核心手段

function debounce(fn, delay) {
  let timer = null;
  return function(...args) {
    clearTimeout(timer);
    timer = setTimeout(() => {
      fn.apply(this, args);
    }, delay);
  };
}

timer 能跨多次函数调用存在，本质原因正是闭包。

1.2.4 垃圾回收与内存泄漏风险

1. JavaScript 的垃圾回收机制

现代 JS 引擎（如 V8）主要采用：

标记-清除（Mark & Sweep）算法

  • 从根对象（全局对象、当前执行栈）开始
  • 递归标记所有可达对象
  • 未被标记的对象会被回收

闭包的影响：

  • 被闭包引用的 AO 是“可达对象”
  • 即使函数执行完，也不会被回收

2. 常见内存泄漏场景

1）意外的全局变量

  • 及时解除引用

  • 使用 WeakMap / WeakSet

    • 适合缓存临时对象

  • 组件卸载时清理副作用

    • 定时器
    • 事件监听
    • 订阅
